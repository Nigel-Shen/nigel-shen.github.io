Gradient Descent and Neural Network

Nigel Shen cs5897

1. Recap

In order to create a neural network with multiple layers, we are supposed to add a non-linear function in each hidden neuron layer; otherwise if the function is linear, then the final output is also linear with respect to the input. That said, for a given set of input $x_i$, we can create a hidden neuron $h_j$ by: $$h_j=g\left(\sum_{i=1}^dw_{ij}x_i\right)$$ where $d$ is the number of input, and $g$ is a nonlinear function. In practice, $g$ are often chosen to be the following functions: $\tanh(x)$, $\textrm{sigmoid}(x)=\frac{1}{1+e^{-x}}$, or $\textrm{relu}(x)=\max(0, x)$.

ï¿¼

Taking advantage of the nonlinear functions $g$ in the model, we can predict some highly nonlinear results using simple neuron networks.

2. Gradient Descent for Neural Network Training

2.1 From Hidden Layer to Output

Given the vector of the hidden layer $\textbf{h}$, in order to find the optimal output $y =g\left(\sum_{k}w_{k}h_k\right) $ such that the error function $E=I\frac{1}{2}(t-y)^2$is minimized, we can use gradient descent in the weight space, moving the weight vector $\textbf{w}$ towards the optimum. To do this, we first need to find the derivative of the nonlinear function $g$.

Prerequisite: The Chain Rule
In order to compute derivative of $f(g(x))$, we can use the following rule:

$$\frac{d}{dx}f(g(x))=fâ(g)gâ(x)=\frac{df}{dg}\frac{dg}{dx}$$

As an example, we use the $\textrm{sigmoid}$ function as our nonlinear function in the neural network model. We take the derivative of the $\textrm{sigmoid}(x)$ using the chain rule:
\begin{align*}
\frac{d}{dx}\frac{1}{1+e^{-x}}&=\frac{-1}{(1+e^{-x})^2}\frac{d}{dx}(1+e^{-x}) \\
&=\frac{e^{-x}}{(1+e^{-x})^2}\\
&=\frac{1}{(1+e^{-x})}\left(1-\frac{1}{1+e^{-x}}\right)\\
&=\textrm{sigmoid}(x)(1-\textrm{sigmoid}(x))
\end{align*}
Now that we know the derivative of the $\textrm{sigmoid}$ function, we can compute the derivative of error with respect to each component of $\textbf{w}$. Denote $s=\textbf{w}\cdot \textbf{h} =\sum_{k}w_kh_k$, then $y=\textrm{sigmoid}(s)$ We thus have:
\begin{align*}
\frac{dE}{dw_k}&=\frac{dE}{dy}\frac{dy}{ds}\frac{ds}{dw_k}\\
&=-(t-y) \textrm{sigmoid}(s)(1-\textrm{sigmoid}(s))\frac{d}{dx_k}\sum_kw_kh_k\\
&= -(t-y) y(1-y)h_k\\
\end{align*}
We can then apply gradient descent to the weight vector using the derivatives we calculated, by iterating the following: $$w_k\rightarrow w_k+\alpha\frac{dE}{dw_k}= w_k+\alpha (t-y) y(1-y)h_k $$

ï¿¼

2.2 From Input to Output

Now consider the weight vector $\textbf{w}^k$ that transform input to the hidden layer $h_k$. Similar to above, we can denote $s_k= \textbf{w}^k\cdot \textbf{x}= \sum_{i}w^k_ix_i $, so that $h_k = \textrm{sigmoid}(s_k)$. By the chain rule, we have:
\begin{align*}
\frac{dE}{dw^k_i}&=\frac{dE}{dy}\frac{dy}{ds}\frac{ds}{dh_k}\frac{dh_k}{ds_k}\frac{ds_k}{dw^k_i}\\
&=(y-t)y(1-y)w_kh_k(1-h_k)x_i
\end{align*}In order to optimize $\textbf{w}^k$, we also use gradient descent:

$$w^k_i\rightarrow w^k_i+\alpha\frac{dE}{dw^k_i}= w^k_i+\alpha (y-t)y(1-y)w_kh_k(1-h_k)x_i $$

3. Other Aspects of Training Neural Networks

3.1 Initialization of Weights

There are a number of strategies to initialize the weights. Thus we start with a âbadâ strategy of initialization: Making weights too large or too small. Consider the $\textrm{sigmoid}$ function, and you will find that, if $|x|>>1$, the gradient of $\textrm{sigmoid}(x)$ will be close to zero: $$\lim_{x\rightarrow\infty}\frac{d}{dx}\textrm{sigmoid}(x)=0$$ Consequently, the neural network will converge very slowly.

Instead, we can initialize randomly in a small interval, e.g., $w_k\sim\textrm{Unif}[-0.01,0.01]$. However, if we have a large number of neurons, the sum of all the weighted values will still be very big. In this case we can ânormalizeâ the interval with the number of neurons: $w_k\sim\textrm{Unif}[-0.01/\sqrt{n},0.01/\sqrt{n}]$.

A more detailed strategy is given by Gloria and Bengio (2010):

- For shallow neural networks, use interval $$\left[-\frac{1}{\sqrt{n}}, \frac{1}{\sqrt{n}}\right]$$ where $n$ is the size of the previous layer;


- For deep neural networks, use interval $$\left[-\sqrt{\frac{6}{n_j+n_{j+1}}}, -\sqrt{\frac{6}{n_j+n_{j+1}}}\ \right]$$ where $n_j$ is the size of the previous layer and $n_{j+1}$ is the size of current layer.


3.2 Multi-class Classification

Sometimes we need to classify with multiple classes ($>2$), then we cannot simply return a scalar to represent the probability of the input belonging to one class against the other. Instead, we need a vector $\textbf{y}$ which represents all the probabilities of all the classes. This vector is called the âOne-hot vectorâ. Notice that the sum of all components of $\textbf{y}$ should be $1$, so we introduce a posterior probability distribution by a softmax defined as: $$\textrm{softmax}(y_i)=\frac{e^{y_i}}{\sum_{}e^{y_j}}$$ The predicted class, in line with our intuition, is the output node $y_i$ with the highest value.

3.3 Training Tricks: Mini Batch SGD

For a classical gradient descent method, one starts with the initial guess $\textrm{w}_0$ and iterate over: $$\textrm{w}_{i+1}= \textrm{w}_i-\alpha\nabla f(\textrm{w}_i, D)$$For stochastic gradient descent, we take a few sample datapoints $d$ that satisfy distribution $D$, i.e., $d\sim D$, and iterate the following: $$\textrm{w}_{i+1}= \textrm{w}_i-\alpha\nabla f(\textrm{w}_i, d)$$
For Mini Batch SGD, we âcombineâ these two schemes by sampling a âbatchâ multiple datapoints $d\sim D$, and:

- Forward propagate it through the network, so that we can obtain the loss or error function;


- Backward propagate to find the gradients $\nabla f(\textbf{w}, d)$


After that, we can update our weight function: $$\textrm{w}_{i+1}= \textrm{w}_i-\alpha\nabla f(\textrm{w}_i, d) $$
